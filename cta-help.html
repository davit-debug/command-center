How it works

All ten tools are fully client-side. They fetch a page from the same origin (your repo), parse it in the browser, and run analysis against a built-in heuristics library. Nothing is sent to a server. Recommendations are based on benchmarks from public CRO research (Ahrefs, Semrush, Backlinko, HubSpot, Nielsen-Norman, Chartbeat) plus the agency's own playbook patterns.

The flow is always the same: load a page → run analysis → see suggestions → tick checkboxes for what you accept → either download the modified HTML or copy a change list to paste back to Claude for execution.

Typical workflow

If you're not sure which tool to start with:

  1. Open the Hub. It auto-scans your homepage and gives a 0–100 health score with a top-3 findings list.
  2. If the score is below 70, jump to whichever tool the findings reference (sticky friction → Picker, weak copy → Microcopy, dead-ends → Paths).
  3. For a single-page deep-dive, use the Picker (the original tool with 5-step research and one-click apply).
  4. For a site-wide audit before a stakeholder meeting, use the Report. Print to PDF.

Language support

Every tool detects EN or KA from <html lang="…"> (or the path prefix as fallback). When swapping CTAs:

  • Both data-en and data-ka attributes are updated together — so the live site's language toggle stays consistent.
  • The visible text in the body is set in the page's language.
  • The Microcopy library has Georgian variants for every English variant (curiosity / specificity / risk-reversal / social-proof / urgency).
  • The placement classifier recognizes both English and Georgian section headings (მომსახურება, კონტაქტი, მზად ხარ, etc.).

Tool-by-tool reference

🧭 01. Hub

Entry point. 0–100 health score + top findings + nav to all tools.

Use when: you want a 10-second read on whether a page needs CTA work, or you want to discover which other tool fits the issue.

Auto-scans the path you type, computes a weighted score (penalizes sticky-bar friction, hero imbalance, weak copy), and surfaces the 3–5 most actionable findings. Doesn't make changes — points you at the right specialized tool.

🔬 02. Picker (Research)

The original tool. 5-step research + one-click apply.

Use when: you want a structured per-CTA decision with reasoning, on a single page.

  • Step 1: Inventory + commitment classification (Low/Medium/High)
  • Step 2: Funnel mapping (TOFU/MOFU/BOFU) based on parent id, heading, position
  • Step 3: Industry-benchmark comparison (Ahrefs/Semrush/HubSpot patterns)
  • Step 4: Friction audit (1–5 scale per CTA)
  • Step 5: Recommended swap list with reasoning, one-click "Apply"

Output: download modified HTML or copy textual change list.

✏️ 03. Microcopy Studio

Same destinations, sharper words.

Use when: the CTA destinations are right, but the wording feels generic ("Choose", "Submit", "Click here").

Scores each CTA's copy quality (5 = specific + first-person + action verb + optimal length + non-generic) and shows 5 reworded variants per CTA — one from each copywriting framework. Pick one, write a custom, or keep.

  • Curiosity: opens information gap ("See What's Hurting My Rankings")
  • Specificity: concrete outcome ("Get My 30-Day SEO Plan")
  • Risk-reversal: removes commitment ("Free Audit — No Strings")
  • Social-proof: implies herd ("Join 35+ Ranked Brands")
  • Urgency: scarcity / immediacy ("Reserve My Strategy Call")

🎯 04. Segments

Match each CTA to visitor temperature.

Use when: the page has the right number of CTAs but they all feel the same — and you suspect each section sees a different visitor type.

Classifies each CTA placement as Cold (just landed), Warm (consumed content), Hot (decision stage), or Returning (knows brand). Computes a 0–100 fit score for the current CTA vs. the ideal CTA for that segment.

🗺️ 05. Sitemap Distribution

Multi-page consistency check.

Use when: you suspect the site's CTA strategy is inconsistent across pages (some service pages overly aggressive, blog posts pushing book-call, etc.).

Crawls 10+ pages, builds a site-wide distribution chart and per-page mix table. Surfaces:

  • Sticky bar inconsistencies across pages
  • Blog posts with high-friction CTAs (wrong for educational pages)
  • Pages with zero conversion CTAs
  • Off-pattern mixes (e.g., service page with no high-commitment CTA)

🧪 06. Lab (Visual)

Live iframe preview, click-to-edit.

Use when: you want to see the change before committing — change copy or href and watch the page update instantly.

Loads the page in a sandboxed iframe, outlines every CTA in purple, labels with index. Click any CTA in the preview (or sidebar list) → edit text/href in the right pane → click "Apply to preview" → see it update live. Viewport switcher tests desktop/tablet/mobile rendering.

⚖️ 07. A/B Test Designer

Hypothesis + variants + sample size.

Use when: you don't want to guess between two CTA copies — you want a proper test plan.

For each conversion CTA, generates a complete test brief: hypothesis statement, control vs. treatment, primary metric, secondary metrics, sample size calculated at chosen confidence + power, and days-to-run estimate based on your traffic. Exports a Markdown plan ready for Optimizely / VWO / GTM split URL.

Stats: standard two-proportion z-test approximation. Default 95% confidence, 80% power, 20% relative MDE.

🛤️ 08. Path Mapper

Where does each CTA lead?

Use when: you want to audit the actual visitor journey — is the funnel a path or a maze?

Crawls pages, builds the CTA destination graph. Flags:

  • Dead-ends — CTA leads to a page with zero further CTAs
  • Orphan pages — pages reachable via nav but no CTA in the funnel points there
  • Missing exits — links to pages not in crawl scope (404 risk)
  • External destinations — Calendly, tel:, off-site exits
  • 2-hop loops — A → B → A patterns (sometimes intentional, often a tell)

🌡️ 09. Heatmap Simulator

% of visitors who actually see each CTA.

Use when: you want to know which CTAs are buried.

Renders the page in an iframe at your chosen viewport (desktop / laptop / tablet / mobile), measures each CTA's actual Y position, then estimates visibility using a scroll-depth distribution curve (above-fold = 100%, 25% scroll = ~70% see, 50% = ~50%, 75% = ~30%, 100% = ~20%). Sticky elements get 100% regardless of position.

📊 10. Report Generator

Stakeholder-ready findings.

Use when: you need a single artifact to share with a client, exec, or design lead.

Crawls a list of pages, runs all the analyses above, and lays it out in a clean executive-report format: site-wide health score, "what's working / what needs work" summary, prioritized action plan with severity + effort, per-page breakdown table, methodology footer. Print-optimized (Cmd-P → Save as PDF) or download as HTML.

Scoring & benchmarks

Commitment level classification

MediumLead form, email, ROI calculator3/5
HighCalendly book (popup), contact form4–5/5
NavPhone (tel:), email (mailto:), section nav (Portfolio, Careers, footer text links)1–2/5

Health score formula (0–100)

Starts at 100, subtracts penalties:

  • −25: more than 80% of conversion CTAs are high-commitment
  • −10: more than 60% are high-commitment
  • −15: zero low-friction CTAs on the page
  • −20: sticky bar uses a high-commitment CTA
  • −10: hero has only high-commitment, no low-friction alternative
  • −5 each: weak copy (generic verbs), capped at 3 instances

CTA detection rules

An anchor is treated as a CTA if it matches at least one of:

  • onclick="…openCalendly…" — Calendly popup trigger
  • href="tel:…" or href="mailto:…"
  • href contains seo-audit.html or lead-form.html
  • Has a CTA-style class: ai-cta-animate, call-btn, or cta-btn

And it must also be button-styled (has padding, background, border, or rounded corners) OR be a Calendly trigger. This filters out plain footer text links that happen to point to seo-audit.html.

Section context is determined by walking up to the nearest <h1>/<h2>/<h3> within 200 lines. Parent ID (within 10 lines) takes precedence for placement classification — that's how the tools distinguish #sticky-cta, #mobile-menu, and #main-content.

Sources / rules of thumb

  • Scroll-depth distribution: Nielsen-Norman scroll & attention studies; Chartbeat audience-depth research
  • CTA friction & commitment ladders: HubSpot CRO benchmarks (2024 reports); Ahrefs / Semrush / Backlinko homepage pattern analysis
  • Microcopy frameworks: Joanna Wiebe (Copyhackers) PAS / specificity / risk-reversal patterns
  • Sample-size approximation: two-proportion z-test (Casella & Berger, Statistical Inference)
  • Hormozi VSL framework (referenced for Picker step 5 hypothesis structure)

Privacy & data flow

  • All ten files are marked <meta name="robots" content="noindex, nofollow">. They will not appear in search results.
  • No data is sent to any external server. All fetch() calls are same-origin (your repo).
  • The Calendly script is removed inside the Lab and Heatmap iframes so its popup doesn't interfere with the preview.
  • Downloads are generated via in-browser Blob URLs. No upload happens.
  • Clipboard usage (for "Copy change list" buttons) requires permission grant by the browser per action.
Calibration note: The benchmarks built into these tools are defaults from public CRO literature. Once you have 6+ months of Calendly booking data + audit-form submission data, the recommendations should be re-calibrated against your own conversion rates. Treat the current numbers as a starting point, not a verdict.
